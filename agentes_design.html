AgentES Minimal Design Specification

一份面向工程实现的 Agent Experience Store 设计方案。本文删除讨论过程中的冗余内容，保留背景、目标、核心抽象、数据模型、CLI、目录结构、实现计划和验收标准，便于程序员直接开始编码。

1. 背景与问题

当前 coding agent、research agent、DevOps agent 等工具型 agent 已经能够执行复杂任务：读取文件、调用 shell、修改代码、运行测试、总结结果。但这些执行过程大多是一次性的。agent 完成任务后，历史上下文通常只以聊天记录、日志、临时文件或不可结构化的 memory 形式存在。

这种方式带来几个问题：

  • 历史经验难以复用。agent 下次遇到类似问题时，不知道过去做过什么。
  • 经验缺乏证据。即使保存了总结，也很难判断它是否真实、是否被验证过。
  • 经验容易误用。过去成功的方法可能只适用于特定项目、工具链或环境。
  • 失败经验没有被系统化保留。agent 会重复尝试已经失败过的策略。
  • 普通 RAG 或向量记忆只解决“相似文本检索”，没有解决“经验能不能安全复用”。

AgentES 的目标是为 agent 提供一个最小但完整的经验存储层，让 agent 不只是保存历史，而是能够将历史转化为有证据、有边界、可检索、可验证、可复用的经验。

2. AgentES 定位

AgentES = Agent Experience Store。 它是一个本地优先、证据感知、由 skill 引导的 agent 经验存储系统。

AgentES 不是底层 POSIX 文件系统，也不是单纯的向量数据库。它位于 agent runtime 之上，提供语义层经验管理能力。

Agent Runtime / Coding Agent / Research Agent
                │
                ▼
AgentES: Experience, Evidence, Trace, Skill, Reuse Protocol
                │
                ▼
Local Filesystem + SQLite + JSONL/YAML/Markdown

AgentES 要支持的核心闭环是：

agent 执行任务
→ 记录 run / trace / evidence
→ 提炼 experience
→ 下次任务搜索相关 experience
→ 检查 evidence 和 reuse boundary
→ validate-use
→ 复用
→ 写回复用结果

AgentES 的最小版本重点不是“存很多东西”，而是把经验复用协议跑通。

3. MVP 范围

3.1 MVP 要做什么

  • 提供本地目录 .agentes/ 作为经验仓库。
  • 使用 SQLite 存储 metadata、索引、关系和搜索表。
  • 使用 YAML / Markdown / JSONL 存储可读对象。
  • 支持 Run、Trace、Evidence、Experience、Skill 五类对象。
  • 支持经验导入、搜索、打开、适用性验证、适配 checklist、复用记录。
  • 提供一个默认 agent skill，告诉 agent 如何使用 AgentES。

3.2 MVP 不做什么

第一版不要做 FUSE、分布式、多租户、复杂权限、自动 redaction、图数据库、复杂 vector DB、Web UI、自动 pattern promotion 或 skill 自动演化。

MVP 推荐实现为：

Python + Typer CLI + SQLite FTS5 + Pydantic + YAML + JSONL + local filesystem

4. 设计原则

4.1 经验必须证据化

没有 evidence 的内容不能成为高置信经验。agent 的自我总结不是证据，命令结果、测试结果、diff、metric、人类反馈才是证据。

4.2 经验必须有复用边界

每条 experience 必须包含：

applies_when
avoid_when
required_checks
validation_after_reuse

4.3 区分观察、假设和验证事实

经验里必须区分：

observations: 看到的现象
hypotheses: 可能原因
verified_facts: 有 evidence 支撑的事实

4.4 失败经验同样重要

失败、警告、废弃经验应该被保留，用来避免重复试错。

4.5 渐进式读取

agent 默认不应该读取所有 raw trace，而应该按层读取：

search card → summary → reuse boundary → evidence → trace slice → raw trace

4.6 经验复用前必须 validate-use

agent 不能因为文本相似就照搬经验。必须先检查当前 context 是否满足 applies_when，是否触发 avoid_when，required_checks 是否完成。

5. 核心对象

对象 用途 MVP 存储形式
Run 一次 agent 任务执行，记录任务、环境、状态、输入输出。 YAML manifest + SQLite metadata
Trace 一次 run 的步骤级结构化日志。 JSONL
Evidence 支撑经验结论的证据，例如测试通过、命令结果、diff。 YAML manifest + blob files
Experience 可复用经验，包含问题、行动、结果、诊断、复用边界和证据引用。 YAML manifest + Markdown views + SQLite FTS
Skill 告诉 agent 如何搜索、验证、复用和写回经验。 Markdown

设计建议：

  • SQLite 负责查询、索引和关系。
  • 文件系统负责保存可读对象和大文本。
  • 经验对象既要 machine-readable，也要 human-readable。

7. 数据模型与 Schema

7.1 SQLite Tables

runs

CREATE TABLE runs (
  id TEXT PRIMARY KEY,
  task_type TEXT,
  task_summary TEXT,
  status TEXT,
  project TEXT,
  repo TEXT,
  started_at TEXT,
  finished_at TEXT,
  manifest_path TEXT
);

traces

CREATE TABLE traces (
  id TEXT PRIMARY KEY,
  run_id TEXT,
  path TEXT,
  created_at TEXT,
  FOREIGN KEY(run_id) REFERENCES runs(id)
);

evidence

CREATE TABLE evidence (
  id TEXT PRIMARY KEY,
  run_id TEXT,
  type TEXT,
  claim TEXT,
  strength TEXT,
  path TEXT,
  created_at TEXT,
  FOREIGN KEY(run_id) REFERENCES runs(id)
);

experiences

CREATE TABLE experiences (
  id TEXT PRIMARY KEY,
  status TEXT,
  confidence TEXT,
  task_type TEXT,
  domain TEXT,
  project TEXT,
  repo TEXT,
  title TEXT,
  summary TEXT,
  problem TEXT,
  diagnosis TEXT,
  applies_when TEXT,
  avoid_when TEXT,
  required_checks TEXT,
  evidence_count INTEGER,
  created_at TEXT,
  updated_at TEXT,
  last_validated_at TEXT,
  manifest_path TEXT
);

experience_evidence

CREATE TABLE experience_evidence (
  experience_id TEXT,
  evidence_id TEXT,
  PRIMARY KEY (experience_id, evidence_id),
  FOREIGN KEY(experience_id) REFERENCES experiences(id),
  FOREIGN KEY(evidence_id) REFERENCES evidence(id)
);

reuse_events

CREATE TABLE reuse_events (
  id TEXT PRIMARY KEY,
  experience_id TEXT,
  run_id TEXT,
  result TEXT,
  notes TEXT,
  created_at TEXT,
  FOREIGN KEY(experience_id) REFERENCES experiences(id),
  FOREIGN KEY(run_id) REFERENCES runs(id)
);

experience_fts

CREATE VIRTUAL TABLE experience_fts USING fts5(
  experience_id,
  title,
  summary,
  problem,
  diagnosis,
  applies_when,
  avoid_when,
  required_checks
);

8. CLI 设计

推荐命令名为 agentes。CLI 是 agent 和程序员都能使用的主接口。

8.1 初始化

agentes init

创建 .agentes/ 目录、SQLite schema、默认 skill。

8.2 Run

agentes run start \
  --task-type code_debugging \
  --summary "Fix import error after schema update" \
  --project demo \
  --repo demo-repo

agentes run finish run_20260428_001 --status success

8.3 Trace

agentes trace add run_20260428_001 \
  --type command \
  --command "pnpm test" \
  --exit-code 1 \
  --summary "Tests failed with missing generated client" \
  --stdout stdout.txt \
  --stderr stderr.txt

8.4 Evidence

agentes evidence create run_20260428_001 \
  --type command_result \
  --claim "pnpm test passed after generation" \
  --strength strong \
  --command "pnpm test" \
  --exit-code 0 \
  --stdout stdout.txt \
  --stderr stderr.txt

8.5 Experience Import

agentes experience import examples/exp_generated_artifact.yaml

第一版只需要支持从 YAML 导入 experience。agent 可以生成 YAML，然后调用 import。

8.6 Experience Search

agentes experience search \
  --query "import error generated client schema update" \
  --task-type code_debugging \
  --status success \
  --min-confidence low

输出 compact search cards，包括 id、title、status、confidence、evidence_count、下一步建议。

8.7 Experience Open

agentes experience open exp_20260428_001 --summary
agentes experience open exp_20260428_001 --reuse
agentes experience open exp_20260428_001 --evidence
agentes experience open exp_20260428_001 --full

8.8 Validate Use

agentes experience validate-use exp_20260428_001 \
  --context examples/context_generated_artifact.yaml

输出 experience 对当前 context 的适用性、风险、匹配条件、未完成检查项和建议。

8.9 Adapt

agentes experience adapt exp_20260428_001 \
  --context examples/context_generated_artifact.yaml

把经验转换成当前任务下的 checklist。agent 应该使用 checklist，而不是直接复制历史命令。

8.10 Reuse Record

agentes reuse record \
  --experience exp_20260428_001 \
  --run run_20260428_002 \
  --result success \
  --notes "Worked after adapting generator command"

8.11 Skill

agentes skill open global_experience_retrieval

9. Agent 使用流程

AgentES 的标准使用流程：

  1. agent 接收任务。
  2. 创建当前 run。
  3. 记录关键 trace。
  4. 搜索相关成功经验和失败经验。
  5. 打开候选经验的 reuse 和 evidence。
  6. 使用 validate-use 判断适用性。
  7. 使用 adapt 生成本地 checklist。
  8. agent 执行任务并验证结果。
  9. 记录 reuse result。
  10. 如果产生新经验，写 experience YAML 并 import。
agentes init

RUN=$(agentes run start \
  --task-type code_debugging \
  --summary "Fix generated client import error" \
  --project demo \
  --repo demo-repo)

agentes experience search \
  --query "import error generated client schema update" \
  --task-type code_debugging

agentes experience open exp_generated_artifact --reuse
agentes experience open exp_generated_artifact --evidence

agentes experience validate-use exp_generated_artifact \
  --context current_context.yaml

agentes experience adapt exp_generated_artifact \
  --context current_context.yaml

# agent acts and validates locally

agentes reuse record \
  --experience exp_generated_artifact \
  --run "$RUN" \
  --result success

10. Skill 设计

默认 skill 应该放在 .agentes/objects/skills/global_experience_retrieval.md。它告诉 agent 如何使用 AgentES。

# Skill: Using AgentES

AgentES stores evidence-backed experiences from prior agent runs.

## Rules

1. Do not reuse an experience based only on title or summary.
2. Before using an experience, inspect reuse and evidence.
3. Search negative or failed experiences when planning a risky strategy.
4. Validate applicability with `agentes experience validate-use`.
5. Convert experience into a checklist; do not copy old commands blindly.
6. After use, record the outcome with `agentes reuse record`.
7. If a new reusable lesson was learned, create an experience YAML and import it.

## Standard Flow

1. Classify the task.
2. Search experiences:

```bash
agentes experience search --query "<task symptoms>" --task-type "<type>"
```

3. Open relevant experience:

```bash
agentes experience open <id> --reuse
agentes experience open <id> --evidence
```

4. Validate:

```bash
agentes experience validate-use <id> --context current_context.yaml
```

5. Adapt:

```bash
agentes experience adapt <id> --context current_context.yaml
```

6. Act and validate locally.

7. Record reuse:

```bash
agentes reuse record --experience <id> --run <run_id> --result success|failure|partial
```

这个 skill 是 AgentES 与普通文件夹或 RAG 的关键区别：它强制 agent 按协议使用经验，而不是只做文本检索。

13. 验收标准

当下面这段流程可以跑通，Minimal AgentES 就达到第一版验收标准：

agentes init

RUN=$(agentes run start \
  --task-type code_debugging \
  --summary "Fix generated client import error" \
  --project demo \
  --repo demo-repo)

agentes trace add "$RUN" \
  --type command \
  --command "pnpm test" \
  --exit-code 1 \
  --summary "Tests failed with missing generated client"

EV=$(agentes evidence create "$RUN" \
  --type command_result \
  --claim "Tests passed after regenerating generated client" \
  --strength strong \
  --command "pnpm test" \
  --exit-code 0)

agentes experience import examples/exp_generated_artifact.yaml

agentes experience search \
  --query "import error generated client schema update" \
  --task-type code_debugging

agentes experience open exp_generated_artifact --reuse
agentes experience open exp_generated_artifact --evidence

agentes experience validate-use exp_generated_artifact \
  --context examples/context_generated_artifact.yaml

agentes experience adapt exp_generated_artifact \
  --context examples/context_generated_artifact.yaml

agentes reuse record \
  --experience exp_generated_artifact \
  --run "$RUN" \
  --result success
核心验收点：AgentES 必须完成 search → inspect evidence → validate applicability → adapt → record reuse 的闭环。

14. 后续路线

14.1 v1.1

  • 增加失败经验搜索和 --include-negative。
  • 增加 search ranking：evidence_count、confidence、last_validated_at boost。
  • 增加 experience deprecate。
  • 增加 evidence open。

14.2 v1.2

  • 增加 embedding search，但保留 FTS 和 metadata filters。
  • 增加 conflict detection。
  • 增加 trace slice。
  • 增加 sensitivity 字段和简单 redaction 提醒。

14.3 v2

  • Pattern promotion：从多个 experience 归纳 pattern。
  • Skill versioning：skill 可评估、可废弃。
  • Graph index：经验、证据、run、pattern、skill 的关系查询。
  • 与底层 agent filesystem / sandbox / runtime 集成。

14.4 最终方向

AgentES 长期可以发展为 agent 的经验操作层：既能记录执行历史，又能提炼可复用经验，并通过 skill 约束 agent 正确使用经验。

history storage → evidence-backed experience → reusable pattern → skill-guided behavior
